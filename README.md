# Лабораторная работа 1: Классификация изображений с использованием CNN
Выполнена в рамках дисциплины Deep Learning (ITMO)
---

## 1. Назначение и цель работы

Освоить основы построения и обучения свёрточных нейронных сетей (CNN) для задачи классификации изображений. Изучить влияние предобработки, аугментации и архитектурных параметров на качество и обобщающую способность модели.

---

## 2. Часть 1: Введение в CNN для классификации изображений

### 2.1. Цель части

Изучить базовую архитектуру CNN (свёрточные слои, пулинг, активационные функции) и научиться обучать модель классификации на датасете CIFAR-10.

### 2.2. Задачи

| № | Задача |
|---|--------|
| 1 | Загрузить датасет CIFAR-10, выполнить нормализацию, подготовить данные для обучения |
| 2 | Определить архитектуру сети с двумя свёрточными слоями, пулингом и полносвязными слоями |
| 3 | Реализовать тренировочный цикл, запустить обучение, отслеживать точность на обучающей и тестовой выборках |
| 4 | Визуализировать функцию потерь и точность, предложить улучшения для повышения производительности |

### 2.3. Ожидаемые результаты

- Обученная модель CNN на CIFAR-10
- Графики loss и accuracy
- Выводы о возможности улучшения архитектуры

---

## 3. Часть 2: Предобработка и аугментация данных

### 3.1. Цель части

Понять влияние предобработки и аугментации данных на работу CNN. Изучить, как различные методы помогают модели обобщать на разных датасетах.

### 3.2. Датасеты для исследования

| Датасет | Описание |
|---------|----------|
| CIFAR-10 | 10 классов объектов (самолёты, автомобили, птицы и др.) |
| Fashion-MNIST | 10 классов предметов одежды |
| SVHN | цифры на фотографиях домовых номеров |

### 3.3. Методы аугментации

| Метод | Описание |
|-------|----------|
| Горизонтальное отражение | отражение изображения слева направо |
| Повороты | случайный поворот на небольшой угол |
| Изменение яркости | случайное изменение освещения |
| Сдвиги | смещение изображения по горизонтали/вертикали |
| Масштабирование | случайное увеличение/уменьшение |

### 3.4. Задачи

| № | Задача |
|---|--------|
| 1 | Загрузить и предобработать CIFAR-10, Fashion-MNIST, SVHN |
| 2 | Применить различные техники аугментации к каждому датасету |
| 3 | Обучить модель CNN на каждом датасете |
| 4 | Сравнить точность и функцию потерь |
| 5 | Оценить влияние аугментации на обобщающую способность |

### 3.5. Ожидаемые результаты

- Сравнительная таблица точности моделей с аугментацией и без
- Выводы о наиболее эффективных методах для каждого датасета
- Анализ влияния предобработки на обобщение

---

## 4. Часть 3: Оптимизация архитектуры CNN

### 4.1. Цель части

Изучить влияние архитектурных параметров (количество фильтров, размеры ядер, Batch Normalization, Dropout) на производительность модели.

### 4.2. Архитектурные варианты

| Вариант | Количество фильтров | Размер ядер | Batch Normalization | Dropout |
|---------|---------------------|-------------|---------------------|---------|
| A (базовый) | малый | малый | нет | нет |
| B (средний) | средний | средний | да | нет |
| C (глубокий) | большой | большой | да | да |

### 4.3. Задачи

| № | Задача |
|---|--------|
| 1 | Создать три варианта архитектуры CNN с разным количеством фильтров и размеров ядер |
| 2 | Добавить и настроить Batch Normalization |
| 3 | Добавить Dropout для предотвращения переобучения |
| 4 | Сравнить модели по точности и функции потерь |

### 4.4. Ожидаемые результаты

- Сравнение трёх архитектур (графики loss, accuracy)
- Выводы о влиянии Batch Normalization на сходимость
- Выводы о влиянии Dropout на переобучение
- Рекомендации по выбору архитектуры

---

## 5. Часть 4: Оптимизация гиперпараметров

### 5.1. Цель части

Изучить влияние гиперпараметров (learning rate, batch size, количество слоёв) и применить нестандартные методы оптимизации: циклический темп обучения, адаптивный learning rate, накопление градиентов.

### 5.2. Исследуемые параметры

| Параметр | Варианты |
|----------|----------|
| Learning rate | 1e-2, 1e-3, 1e-4 |
| Batch size | 16, 32, 64, 128 |
| Количество слоёв | 2, 3, 4 свёрточных слоя |

### 5.3. Методы оптимизации

| Метод | Описание |
|-------|----------|
| Cyclic Learning Rate (CLR) | циклическое изменение learning rate в заданном диапазоне |
| Adaptive Learning Rate | автоматическая подстройка LR (ReduceLROnPlateau) |
| Gradient Accumulation | накопление градиентов для экономии памяти при большом batch size |

### 5.4. Задачи

| № | Задача |
|---|--------|
| 1 | Обучить модель с циклическим темпом обучения |
| 2 | Обучить модель с адаптивным learning rate (ReduceLROnPlateau) |
| 3 | Протестировать разные значения batch size с накоплением градиентов |
| 4 | Сравнить модели с разным количеством свёрточных слоёв |
| 5 | Предложить оптимальные настройки гиперпараметров |

### 5.5. Ожидаемые результаты

- Сравнение стратегий управления learning rate
- Анализ влияния batch size на сходимость и использование памяти
- Оптимальная конфигурация гиперпараметров для повышения точности и стабильности обучения

## 9. Примечания

- Все части работы выполняются последовательно.
- Для аугментаций рекомендуется использовать `torchvision.transforms`.
- Для циклического learning rate использовать `torch.optim.lr_scheduler.CyclicLR`.
- Для адаптивного LR использовать `ReduceLROnPlateau`.
- При ограниченных ресурсах допускается уменьшение количества эпох, но с сохранением сравнительного анализа.
